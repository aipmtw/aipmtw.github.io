# SOP：首頁資料同步

## 目的

確保 aipm.com.tw 首頁顯示的資訊與各 repo 實際狀態一致，包括卡片資料、巡檢結果、需求紀錄等。

## 現況架構（Supabase 資料庫驅動）

所有資料存放在 Supabase 資料庫，前端透過 Supabase JS Client 即時讀取：

| 資料表 | 說明 | 更新時機 |
|--------|------|---------|
| `repos` | Repo 基本資料（名稱、URL、顏色、commit 資訊、issue 數量） | 每次巡檢同步 |
| `issues` | 各 repo 最近已完成 issue 紀錄 | 每次巡檢更新 |
| `check_results` | 巡檢摘要（時間、總結） | 每次巡檢新增 |
| `check_details` | 巡檢各 repo 詳情 | 每次巡檢新增 |

### Supabase 專案資訊

- URL：`https://reipdepbltfbfxnjjegy.supabase.co`
- 認證檔：`$BASE_DIR/2026biz/.env.supabase`
- 前端使用 publishable key（公開讀取）
- 巡檢腳本使用 service key（寫入權限）

## 同步項目

### 1. Repo 資訊（repos 表）

每次巡檢 UPSERT 到 `repos` 表：
- **last_commit_hash**：最新 commit 短 hash（7 碼）
- **last_commit_time**：commit 時間（ISO 格式）
- **valid_issue_count**：Collaborator 提交的已關閉 issues 總數

### 2. 新增 Repo 偵測

- 比對 public repo 清單與 `repos` 表中 `is_active = true` 的記錄
- 排除系統 repo（如 space-template）
- 自動 INSERT 新 repo 到 `repos` 表

### 3. 刪除 Repo 清理

- 用 `gh api repos/aipmtw/{name}` 檢查是否存在
- 404 → UPDATE `repos` SET `is_active = false`

### 4. 需求紀錄（issues 表）

- 查詢每個 repo 最近 3 筆已關閉 issues
- UPSERT 到 `issues` 表（repo_id, issue_number, author, summary, completed_at）

### 5. 巡檢結果（check_results + check_details 表）

- 每次巡檢 INSERT 一筆 `check_results`（時間 + 摘要）
- 再 INSERT 各 repo 的 `check_details`（open_issues + result）

## 寫入方式

巡檢腳本使用 curl 呼叫 Supabase REST API：

```bash
source $BASE_DIR/2026biz/.env.supabase

# UPSERT repos
curl -s -X POST "$SUPABASE_URL/rest/v1/repos" \
  -H "apikey: $SUPABASE_SERVICE_KEY" \
  -H "Authorization: Bearer $SUPABASE_SERVICE_KEY" \
  -H "Content-Type: application/json" \
  -H "Prefer: resolution=merge-duplicates" \
  -d '{"id": "...", "last_commit_hash": "...", ...}'

# INSERT check_results
curl -s -X POST "$SUPABASE_URL/rest/v1/check_results" \
  -H "apikey: $SUPABASE_SERVICE_KEY" \
  -H "Authorization: Bearer $SUPABASE_SERVICE_KEY" \
  -H "Content-Type: application/json" \
  -H "Prefer: return=representation" \
  -d '{"check_time": "...", "summary": "..."}'
```
