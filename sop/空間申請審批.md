# SOP：空間申請與刪除審批

## 目的

管理用戶申請新空間或刪除既有空間的完整流程，從申請到自動化執行。

## 申請空間流程

```
用戶填寫表單 → 跳轉 GitHub 建 Issue → 管理員 /approve → GitHub Actions 自動建立
```

### 1. 用戶申請

1. 在 https://aipm.com.tw/ 底部填寫空間名稱和 GitHub 帳號
2. 系統即時驗證：
   - 名稱格式（小寫英文+數字+連字號，2-32 字元）
   - 名稱是否可用（不重複、不在待審批中）
   - GitHub 帳號是否存在
   - 是否已達上限（每人最多 2 個空間）
3. 驗證通過後，系統產生 SHA-256 簽章（salt: `aipmtw-2026`）
4. 跳轉至 GitHub Issue 預填頁面，**用戶自己登入 GitHub 送出**

### 2. 管理員審批

1. 收到 Issue 通知（標題：`[申請空間] {名稱} - {帳號}`）
2. 確認申請合理性
3. 在 Issue 留言 `/approve`

### 3. 自動執行（GitHub Actions）

`create-space.yml` 自動：
1. 驗證簽章完整性
2. 從 `space-template` 建立新 repo
3. 替換模板中的 `SPACE_NAME` 為實際名稱
4. 安裝依賴（`npm install`）
5. 推送程式碼
6. 將申請人加為 Collaborator（push 權限）
7. 設定 GitHub Pages（workflow build type）
8. 關閉 Issue 並留言通知

### 4. 巡檢自動同步

下次 `/do-issues` 執行時，自動偵測新 repo 並新增首頁卡片。

## 刪除空間流程

```
用戶填寫表單 → 跳轉 GitHub 建 Issue → 管理員 /approve → GitHub Actions 自動刪除
```

### 步驟

1. 在首頁填寫要刪除的空間名稱和 GitHub 帳號
2. 系統驗證空間存在、帳號有效，產生簽章（salt: `aipmtw-2026-delete`）
3. 跳轉至 GitHub Issue 頁面送出
4. 管理員留言 `/approve`
5. `delete-space.yml` 自動刪除 repo 並關閉 Issue
6. 下次巡檢自動移除首頁卡片

## 簽章驗證

| 類型 | Salt | 演算法 |
|------|------|--------|
| 申請空間 | `aipmtw-2026` | SHA-256(`salt\|name\|account`) 取前 16 字元 |
| 刪除空間 | `aipmtw-2026-delete` | SHA-256(`salt\|name\|account`) 取前 16 字元 |

簽章用途：防止 Issue 內容被竄改，確保名稱和帳號與表單填寫一致。

## 管理員審批檢查清單

### 申請空間

- [ ] 空間名稱是否合適（不含不當用語）
- [ ] GitHub 帳號是否為真人用戶
- [ ] 該用戶是否已有 2 個空間
- [ ] 簽章是否完整（Issue body 未被修改）

### 刪除空間

- [ ] 確認申請人是否為空間擁有者或 Collaborator
- [ ] 確認空間內容是否已備份（如需要）
- [ ] 簽章是否完整

## 注意事項

- Issue 必須由申請人本人建立（不可代建），以確保作者身份可追溯
- 巡檢系統不會自動關閉申請/刪除類 Issue，必須等管理員 `/approve`
- 若 Issue 內容被修改，簽章驗證將失敗，GitHub Actions 不會執行
