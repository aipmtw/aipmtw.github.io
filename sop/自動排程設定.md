# SOP：自動排程設定

## 目的

設定 `/do-issues` 和 `/check-actions` 的自動排程，實現無人值守的持續巡檢。

## 排程方式

### 方式一：Session 內排程（臨時）

在 Claude Code 互動模式中：

```
/loop 15m /do-issues
```

- 優點：簡單快速
- 限制：session 結束即失效，最多 3 天自動過期

### 方式二：Mac cron（持久）

```bash
# 每 15 分鐘，在 :05, :20, :35, :50 執行
crontab -e
# 加入：
5,20,35,50 * * * * cd ~/2026aipm-projects/2026biz && claude -p "/do-issues" --dangerously-skip-permissions >> ~/2026aipm-projects/scheduler.log 2>&1
```

### 方式三：Windows 工作排程器（持久）

1. 建立批次檔 `C:\2026aipm-projects\do-issues.bat`：
   - 含碰撞偵測（lock file、檢查是否已在執行）
   - 含逾時自動清除（30 分鐘）
2. 用 PowerShell 建立排程工作

## 關鍵參數

| 參數 | 說明 |
|------|------|
| `--dangerously-skip-permissions` | 跳過所有權限確認，無人值守必要 |
| `-p` | 非互動模式，執行完自動退出 |

## 碰撞防護

排程執行時需防止多個 instance 同時執行：

1. **Lock file**：`$BASE_DIR/.claude/scheduled_tasks.lock`
2. **Process 檢查**：確認沒有其他 claude 在同目錄執行
3. **逾時清除**：lock file 超過 30 分鐘自動清除

## 管理員操作

| 操作 | 方式 |
|------|------|
| 查看排程狀態 | `crontab -l`（Mac）或工作排程器（Windows） |
| 暫停排程 | 註解 cron 行或停用排程工作 |
| 查看執行日誌 | `tail -f ~/2026aipm-projects/scheduler.log` |
| 手動觸發 | `claude -p "/do-issues" --dangerously-skip-permissions` |
