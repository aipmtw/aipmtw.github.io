# 自動建立空間與預設首頁 — 規劃書 (Take 2)

> 基於 0314.md，移除 Email 欄位，簡化為兩個欄位即可申請。

## 目標

讓新成員透過 aipm.com.tw 首頁填寫基本資料，經審批後自動建立個人空間（GitHub repo + 預設網站）。

## 申請表單欄位（2 個）

1. **空間名稱**（即 repo 名稱）
   - 驗證：即時呼叫 `gh api repos/aipmtw/{name}` 檢查是否已存在
   - 格式：小寫英文 + 數字 + 連字號，2-30 字元
   - ✅ 名稱可用 → 打勾
   - ❌ 名稱已被使用 或 格式不符 → 顯示提示

2. **GitHub Account**
   - 驗證：即時呼叫 `GET https://api.github.com/users/{username}` 確認帳號存在
   - ✅ 帳號存在 → 打勾，顯示 GitHub 頭像確認
   - ❌ 帳號不存在 → 顯示提示

## 流程

```
申請人填表 → 兩個都打勾 → 點擊「提交申請」
    ↓
自動建立 GitHub Issue（標題：[申請空間] {空間名稱} - {GitHub Account}）
    ↓
管理員（aipmtw）收到 Issue 通知
    ↓
管理員在 Issue 留言 /approve
    ↓
GitHub Actions 自動執行建立流程
    ↓
完成 → Issue 自動關閉，申請人收到 GitHub 通知
```

## 前端實作

### 申請表單 UI

在 aipm.com.tw 首頁，卡片區域下方新增「申請空間」區塊：

```html
┌─────────────────────────────────────┐
│        申請你的個人空間               │
│                                     │
│  空間名稱  [____________] ✅         │
│  GitHub    [____________] ✅ 👤      │
│                                     │
│         [ 提交申請 ]                 │
│  （兩個都打勾才可點）                 │
└─────────────────────────────────────┘
```

### 驗證邏輯（前端 JavaScript）

1. **空間名稱驗證**（輸入停止 500ms 後觸發）：
   ```javascript
   // 格式檢查
   /^[a-z0-9][a-z0-9-]{0,28}[a-z0-9]$/.test(name)
   // 可用性檢查
   fetch(`https://api.github.com/repos/aipmtw/${name}`)
     .then(r => r.status === 404 ? '可用' : '已存在')
   ```

2. **GitHub 帳號驗證**（輸入停止 500ms 後觸發）：
   ```javascript
   fetch(`https://api.github.com/users/${username}`)
     .then(r => r.ok ? r.json() : null)
     .then(data => data ? { valid: true, avatar: data.avatar_url } : { valid: false })
   ```

3. **提交**（兩個都通過才啟用按鈕）：
   ```javascript
   // 透過 GitHub API 建立 Issue
   // 需要用戶授權或使用 public repo 的 issue 建立
   ```

### 提交方式：跳轉 GitHub Issue 預填表單（由申請人本人送出）

**重要設計決策**：Issue 必須由申請人自己的 GitHub 帳號建立，不可由系統代建。這樣 Issue 作者 = 實際申請人，確保可追溯身份。

使用 GitHub Issue 預填 URL：

```
https://github.com/aipmtw/aipmtw.github.io/issues/new?
  title=[申請空間] {spaceName} - {githubAccount}&
  body=空間名稱：{spaceName}%0AGitHub：{githubAccount}%0A驗證狀態：✅ 名稱可用 ✅ 帳號存在&
  labels=申請空間
```

申請人點擊後跳轉到 GitHub 建立 Issue（需要 GitHub 帳號登入）。

## GitHub Actions 自動化（`.github/workflows/create-space.yml`）

觸發條件：Issue comment 包含 `/approve`，且 Issue 標題以 `[申請空間]` 開頭。

```yaml
on:
  issue_comment:
    types: [created]

jobs:
  create-space:
    if: |
      github.event.comment.body == '/approve' &&
      github.event.comment.user.login == 'aipmtw' &&
      startsWith(github.event.issue.title, '[申請空間]')
    runs-on: ubuntu-latest
    steps:
      # 1. 解析 Issue 標題取得空間名稱和 GitHub Account
      # 2. 建立 repo
      # 3. 加入協作者
      # 4. 建立預設網站（React + Tailwind）
      # 5. 設定 GitHub Pages
      # 6. 關閉 Issue 並留言通知
```

### 自動化詳細步驟

1. **解析 Issue**：從標題 `[申請空間] {name} - {account}` 提取參數
2. **建立 repo**：`gh repo create aipmtw/{name} --public --description "Member space for {account}"`
3. **Clone & 初始化**：
   - 建立 Vite + React + Tailwind CSS 專案
   - 設定 `vite.config.js` 的 `base: '/{name}/'`
   - 首頁顯示空間名稱 + 返回 aipm.com.tw 連結
   - 建立 `.github/workflows/deploy.yml`
   - 新增 SVG favicon（首字母）
4. **加入協作者**：`gh api repos/aipmtw/{name}/collaborators/{account} -X PUT -f permission=push`
5. **設定 Pages**：`gh api repos/aipmtw/{name}/pages -X POST -f build_type=workflow`
6. **Commit & Push**
7. **更新 aipm.com.tw**：在 index.html 新增卡片（可由下次 do-issues 處理）
8. **關閉 Issue**：留言通知「空間已建立：https://aipm.com.tw/{name}/」

## 實作順序

1. **Step 1**：在 index.html 新增申請表單 UI + 前端驗證邏輯
2. **Step 2**：建立 `.github/workflows/create-space.yml` 自動化

4. **Step 4**：清理測試資料，正式上線
