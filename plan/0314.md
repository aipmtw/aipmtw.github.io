# 自動建立空間與預設首頁 — 規劃書

## 目標

讓新成員透過 aipm.com.tw 首頁填寫基本資料，經審批後自動建立個人空間（GitHub repo + 預設網站）。

## 申請表單欄位

1. **空間名稱**（即 repo 名稱）
   - 驗證：檢查 aipmtw 組織下是否已存在同名 repo
   - 格式：小寫英文 + 數字 + 連字號，2-30 字元
   - ✅ 可用 → 打勾

2. **申請人 Email**
   - 驗證：格式驗證（前端 regex）
   - 確認方式：送出後系統發一封驗證信，申請人點擊信中連結完成驗證
   - ✅ 已驗證 → 打勾

3. **GitHub Account**
   - 驗證：呼叫 GitHub API `GET https://api.github.com/users/{username}` 確認帳號存在
   - ✅ 帳號存在 → 打勾

## 審批流程

1. 申請人填完三個欄位，三個都打勾後，進入「待審批」狀態
2. 管理員（aipmtw）收到通知（GitHub Issue 或 Email），在後台點擊「核准」
3. 核准後自動執行以下動作

## 自動化動作（核准後）

1. **建立 repo**：`gh repo create aipmtw/{空間名稱} --public`
2. **加入協作者**：`gh api repos/aipmtw/{空間名稱}/collaborators/{GitHub Account} -X PUT`
3. **建立預設網站**：
   - 用 Create React App 或 Vite 建立基本 React + Tailwind CSS 專案
   - 首頁只顯示 repo 名稱，含返回 aipm.com.tw 連結
   - 設定 GitHub Pages（workflow 模式）
   - 建立 `.github/workflows/deploy.yml`
4. **Commit & Push**：自動提交初始程式碼
5. **更新 aipm.com.tw**：
   - 在 index.html 新增該成員的卡片
   - 更新 check-issues / check-actions 的 repo 清單

## 後端需求

### 方案 A：GitHub Issues 驅動（無需額外後端）

- 申請表單提交後，自動在 aipmtw.github.io repo 建立一個 Issue，標題格式：`[申請空間] {空間名稱} - {GitHub Account}`
- Issue body 包含三個欄位的驗證狀態
- 管理員在 Issue 留言 `/approve` 觸發自動化
- 自動化由 GitHub Actions workflow 執行

**優點**：不需要額外伺服器
**缺點**：Email 驗證需要額外服務

### 方案 B：Cloudflare Workers（輕量後端）

- 用 Cloudflare Workers 處理：
  - 表單提交 API
  - Email 發送驗證信（透過 Resend / SendGrid）
  - GitHub API 呼叫（repo 建立、協作者邀請）
- Cloudflare KV 儲存申請狀態

**優點**：完整流程自動化
**缺點**：需要 Cloudflare 帳號與設定

### 建議：方案 A 為主

先用方案 A 快速實現，Email 驗證改為「管理員人工確認」（簡化流程）：
1. 前端表單驗證空間名稱可用性 + GitHub 帳號存在性（兩個即時打勾）
2. Email 僅做格式驗證（前端打勾）
3. 提交後建立 GitHub Issue
4. 管理員確認 Email 正確後留言 `/approve`
5. GitHub Actions 自動執行建立流程

## 前端實作

在 aipm.com.tw 首頁新增「申請空間」區塊：
- 三個輸入欄位 + 即時驗證打勾
- 提交按鈕（三個都打勾才可點）
- 提交後跳轉至對應的 GitHub Issue
